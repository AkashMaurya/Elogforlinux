Date Restrictions Management

Configure when students can submit logs and doctors can review them

How Date Restrictions Work

Student Restrictions: Control when students can submit their log entries (past/future date limits and allowed days).

Doctor Restrictions: Control when doctors can review student logs (review deadlines and allowed days).

Review Period: Set automatic deadlines for doctors to review logs after submission.

Allowed Days: Restrict submissions/reviews to specific days of the week (e.g., weekdays only).

Student Log Submission Restrictions

These settings control when students can submit their log entries. Use these to prevent backdating or future dating of logs.

Example: If set to 7, students can only submit logs for dates within the last 7 days. Set to 0 to allow only today's date.

Recommended: Keep disabled unless students need to pre-schedule logs for planned activities.

Days before deadline to send notification to doctors

When the review period is enabled, doctors will have a limited time to review student logs. After the deadline passes, they will no longer be able to review those logs. Notifications will be sent to doctors before the deadline to remind them to complete their reviews.

To send notifications, you need to set up a scheduled task to run the following command: python manage.py send_deadline_notifications

Attendance Tracking Settings

Student Attendance Tracking

Allow doctors to track student attendance during training sessions at various training sites.

When enabled, doctors can access attendance tracking features to mark student presence/absence during training sessions. This includes taking attendance, viewing attendance history, and generating attendance reports.

When disabled, all attendance tracking features will be hidden from doctors and they will receive an error message if they try to access attendance pages.
